## ADDED Requirements

### Requirement: 爬虫健康状态记录
系统 SHALL 在每次爬虫执行后记录其健康状态到 MongoDB，包括成功/失败状态、失败原因、执行时间等信息。

#### Scenario: 爬虫执行成功
- **WHEN** 爬虫成功完成数据抓取
- **THEN** 系统记录成功状态、抓取数据量、执行耗时到健康检查集合

#### Scenario: 爬虫执行失败
- **WHEN** 爬虫执行过程中发生异常
- **THEN** 系统记录失败状态、失败原因分类、错误堆栈、执行耗时到健康检查集合

### Requirement: 健康报告生成
系统 SHALL 提供生成爬虫健康报告的功能，展示各爬虫的成功率、失败原因统计等信息。

#### Scenario: 生成每日健康报告
- **WHEN** 用户请求生成每日健康报告
- **THEN** 系统输出包含所有爬虫当日执行状态的 Markdown 报告

#### Scenario: 生成历史健康趋势
- **WHEN** 用户请求查看爬虫历史健康趋势
- **THEN** 系统展示最近 30 天各爬虫的成功率变化趋势

### Requirement: 失败原因分类
系统 SHALL 将爬虫失败原因分类为：网络错误、解析错误、API 限流、反爬虫、其他。

#### Scenario: 网络超时失败
- **WHEN** 爬虫因网络超时失败
- **THEN** 系统将失败原因分类为"网络错误"

#### Scenario: HTML 解析失败
- **WHEN** 爬虫因无法解析网页结构失败
- **THEN** 系统将失败原因分类为"解析错误"

#### Scenario: API 限流
- **WHEN** 爬虫因 API 返回 429 状态码失败
- **THEN** 系统将失败原因分类为"API 限流"
